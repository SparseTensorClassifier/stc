Documentation¶

class stc.SparseTensorClassifier(targets, features=None, collapse=True, engine='sqlite://', prefix='stc', chunksize=100, cache=True, power=0.5, balance=1, entropy=1, loss='norm', tol=1e-15)¶

The input data must be structured as follows: ‘data’: [{‘key1’: [value1, value2, …, valueN], ‘key2’: [], …, ‘keyN’: []}] Such that each ‘keyN’ is a feature and a dimension of the tensor, while each valueN corresponds to a index position in the tensor structure. The internal sparse representation of the tensor ia a table of the form: v1, v2, …, vn, score where each tuple corresponds to a non zero entry of the tensor along the coordinates given by v1, v2, …, vn The score corresponds to probability up to a normalization factor

Parameters

  • engine (Union[Engine, str]) – connection to the DB

  • features (Optional[List[str]]) – features of the tensor

  • collapse (bool) – merge dimensions. Default True

  • prefix (str) – prefix to use in the DB tables

  • chunksize (int) – number of items per chunk

  • cache (bool) – cache weights to boost performance

  • targets (List[str]) – target variable(s)

  • balance (float) – artificially balance the sample

  • power (float) – probability power

  • loss (str) – loss function. Use ‘norm’ for p-norm(1) loss or ‘log’ for cross-entropy.

  • entropy (float) – use entropic weights

learn(test_size=None, train_size=None, stratify=True, priority=None, max_features=0, max_actions=0, max_iter=1, max_runtime=0, random_state=None)¶

Learn the Best Policy via Reinforcement Learning

Optimize the evaluation metric via cross validation. For each episode, split the train set in train-validation sets. Start with a set of empy features and compute the reward of the state (-loss). Add the value to the Q-table. Explore all the next states generated by adding 1 feature to the empty set. Compute the values of the states. Add to the Q-table. Select the state with the maximum value. Move to that state. Explore all the next states generated by adding 1 feature to the current set of features… Stop when all features are used or when the value of all the next states is less than the value of the current state.

Parameters

  • max_runtime (int) – how long to train the algorithm, in seconds. If 0, no time limit.

  • max_iter (int) – maximum number of iterations to train the algorithm. If 0, no limit.

  • test_size (Optional[float]) – train-test cross validation split

  • train_size (Optional[float]) – train-test cross validation split

  • stratify (bool) – if True, the folds are made by preserving the percentage of samples for each class

  • max_features (int) – number of maximum features to return in the policy. 0 -> no limit

  • max_actions (int) – number of maximum states to explore at once. 0 -> no limit

  • priority (Optional[List[str]]) – list of features to learn first

  • random_state (Optional[bool]) – random number generator seed. Pass an int for reproducible output across function calls.

Return type

Tuple[List[List[str]], List[float]]

Returns

tuple of (best policy, loss)

predict(items, policy=None, probability=True, explain=True)¶

Predict

Use the tensor to predict the data. The target variable(s) can be multidimensional, i.e. predict both variable A and B at the same time. The features to use for predictions are given by the policy. The algorithm starts with the first element of the policy. If it fails, the second element is used, and so on. If the policy ends with the empy list [] then all items will be predicted (eventually using no features, i.e. they will be attributed the most likely class in the trainig set). If not, the prediction may miss for some items.

Parameters

  • items (Union[List[dict], DataFrame]) – the data in JSON or tabular format

  • policy (Optional[List[List[str]]]) – list of lists of features to use for prediction. First lists are applied first

  • explain (bool) – return weights for interpretability

  • probability (bool) – return probability table

Return type

Tuple[DataFrame, Optional[DataFrame], Optional[DataFrame]]

Returns

tuple of (classification, probability, interpretability)
